## 섹션별 상세 내용

### 1. PROBLEM — 해결하려는 문제

| #   | 문제                | 현재 불편함                                                      |
| --- | ------------------- | ---------------------------------------------------------------- |
| 1   | 날짜 취합 비효율    | 카톡 단체방에서 수십 명 의견 취합 → 주최자가 수동 집계           |
| 2   | 참여 장벽           | 구글 폼·Doodle 등은 링크 접속 후에도 절차가 복잡하거나 UI가 낯섦 |
| 3   | 집계 결과 공유 불편 | 취합 후 다시 단체방에 "○○일이 제일 많네요" 메시지 공유 필요      |

### 2. CUSTOMER SEGMENTS — 타겟 고객

| 세그먼트               | 특징                                      | 페인포인트                            |
| ---------------------- | ----------------------------------------- | ------------------------------------- |
| 주최자 (Authenticated) | 직장인 팀장·회식 담당자, Google 계정 보유 | 날짜 취합·집계에 시간 낭비            |
| 게스트 (Anonymous)     | 초대받은 직장 동료, 로그인 불필요         | 복잡한 절차 없이 빠르게 응답하고 싶음 |

### 3. UNIQUE VALUE PROPOSITION — 독보적 가치 제안

> **"링크 하나로 회식 날짜 조율 끝"**
>
> 주최자는 이벤트를 만들고 링크만 공유하면 끝.
> 초대받은 동료는 로그인 없이 이름 입력 후 날짜를 선택.
> 주최자는 실시간 히트맵으로 최적 날짜를 확정한다.

### 4. SOLUTION — 솔루션

| 기능                    | 설명                                             | 대응 문제           |
| ----------------------- | ------------------------------------------------ | ------------------- |
| 이벤트 생성 + 초대 링크 | nanoid 기반 고유 토큰, 클립보드 복사             | 취합 채널 분산      |
| 로그인 없는 게스트 참여 | 이름 입력 → guest_token 발급 → localStorage 저장 | 참여 장벽           |
| 실시간 히트맵           | 날짜별 가능 인원 시각화, Supabase Realtime 구독  | 집계 공유 불편      |
| 재편집 지원             | 재방문 시 localStorage 토큰으로 기존 선택 복원   | 변경 의사 반영 불가 |

### 5. CHANNELS — 채널

| 채널 유형 | 구체 채널                    | 역할                         |
| --------- | ---------------------------- | ---------------------------- |
| 공유      | 카카오톡·슬랙·이메일         | 초대 링크 전파 (입소문 핵심) |
| 커뮤니티  | 직장인 카페, 개발자 커뮤니티 | 초기 사용자 획득             |
| 자연 성장 | 게스트의 재사용              | 게스트 → 주최자 전환         |

### 6. REVENUE STREAMS — 수익 구조

| 단계 | 모델                                           | 목표                   |
| ---- | ---------------------------------------------- | ---------------------- |
| MVP  | 무료                                           | 포트폴리오·사용자 검증 |
| 성장 | 프리미엄 플랜 (이벤트 수 제한 해제, 고급 통계) | 수익화 검토            |
| 확장 | 기업 플랜 (팀 관리, SSO, 우선 지원)            | B2B 수익               |

### 7. COST STRUCTURE — 비용 구조

| 비용 항목     | 내용                               | 규모           |
| ------------- | ---------------------------------- | -------------- |
| 인프라 (고정) | Supabase Free + Vercel Free        | 0원 (MVP)      |
| 인프라 (변동) | Supabase Pro (월간 사용량 초과 시) | $25/월         |
| 개발          | 개인 시간 투자                     | 파트타임 6~8주 |

### 8. KEY METRICS — 핵심 지표

| 지표                    | 측정 방법                | 목표 (MVP) |
| ----------------------- | ------------------------ | ---------- |
| 신규 이벤트 생성 수     | `events` 테이블 COUNT    | 월 50건    |
| 이벤트당 평균 참여자 수 | `participants` JOIN      | 5명 이상   |
| 날짜 확정 전환율        | `status = 'closed'` 비율 | 60% 이상   |
| 게스트 재방문율         | 재편집(F006) 발생 비율   | 20% 이상   |

### 9. UNFAIR ADVANTAGE — 경쟁 우위

| 우위 요소               | 설명                                                    |
| ----------------------- | ------------------------------------------------------- |
| 로그인 없는 게스트 참여 | guest_token 기반으로 마찰을 최소화 — Doodle 대비 차별점 |
| 실시간 히트맵           | Supabase Realtime으로 별도 새로고침 없이 집계 갱신      |
| 재편집 지원             | localStorage 토큰으로 기존 응답 복원·수정 가능          |
| 한국 직장 문화 최적화   | 카카오톡 링크 공유 흐름에 맞춤 설계                     |
